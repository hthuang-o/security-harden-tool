服务器安全加固工具
技术文档

项目概述

服务器安全加固工具是一款用于批量对多个Linux服务器节点进行安全配置检查与加固的自动化运维工具。该工具通过SSH协议批量连接目标服务器，执行预定义的安全检查项和安全加固操作，支持配置回滚功能，确保在执行加固操作后可恢复到原始状态。

本工具采用Python开发，使用Paramiko库实现SSH远程连接，支持并发执行任务，提供命令行和Web界面两种使用方式，适用于企业环境中批量服务器的安全加固场景。

核心功能

1. 节点管理

支持SSH密钥认证和密码认证两种方式，可批量导入多个服务器节点信息。每个节点配置包含主机地址、端口、用户名、认证方式等参数，支持灵活配置。

2. 安全检查

提供四种安全检查类别：

  • 密码策略：检查密码最长使用天数、最短使用天数、最短长度、到期警告天数、密码复杂度等
  • 系统安全：检查空口令账号、UID配置、su权限、日志服务等
  • SSH加固：检查SSH协议版本、认证方式、日志级别、最大重试次数等
  • 文件权限：检查系统关键文件的访问权限配置

3. 安全加固

根据检查结果执行相应的加固命令，可选择执行全部类别或特定类别的加固任务。每次执行前自动备份原始配置，支持回滚操作。

4. 任务回滚

执行加固操作后，可根据备份数据回滚到原始配置状态，支持单个任务回滚和批量回滚。

5. 报告生成

执行完成后自动生成JSON格式的详细报告，包含执行结果、统计信息、合规率等数据。

系统架构

技术栈

  • 后端：Python 3.8+、Flask、Paramiko、PyYAML
  • 前端：Vue 3、Axios、Chart.js
  • 协议：SSH2

模块设计

  • inventory.py：节点配置加载与验证模块
  • ssh_client.py：SSH连接与命令执行模块
  • executor.py：任务执行与回滚逻辑模块
  • tasks/__init__.py：安全任务定义模块
  • app.py：Web服务API模块

安装与运行

环境要求

  • Python 3.8 或更高版本
  • Linux/Unix操作系统

安装步骤

第一步，安装依赖包：

pip install -r requirements.txt

依赖包包括：

  • paramiko：SSH远程连接
  • pyyaml：YAML配置文件解析
  • jinja2：模板引擎
  • flask：Web框架
  • flask-cors：跨域支持

第二步，启动Web服务：

python app.py

服务启动后，默认监听5000端口，访问地址：

  • http://localhost:5000
  • http://127.0.0.1:5000

命令行模式

# 检查配置状态
python main.py -n nodes.yaml --check

# 执行安全加固
python main.py -n nodes.yaml --exec

# 回滚任务
python main.py -n nodes.yaml --rollback --task-name "任务名称"

配置文件说明

nodes.yaml

节点配置文件示例：

nodes:
  - host: 192.168.1.10
    port: 22
    user: root
    auth: key
    key_path: ~/.ssh/id_rsa
    
  - host: 192.168.1.20
    port: 22
    user: root
    auth: password
    password: your_password

配置项说明：

  • host：目标服务器IP地址或主机名
  • port：SSH端口，默认22
  • user：SSH用户名
  • auth：认证方式，key（密钥）或password（密码）
  • key_path：SSH私钥路径
  • password：SSH密码

任务定义（tasks/__init__.py）

安全任务采用字典格式定义，每个任务包含：

  • name：任务名称
  • command：执行命令
  • check：检查命令
  • expected：预期结果
  • backup_key：回滚键名

界面功能实现

1. 节点管理界面

提供节点列表展示、添加删除节点、测试连接等功能。支持单个添加和批量导入两种模式，批量导入格式为：主机地址:端口:用户名:认证方式:密码。

2. 任务执行界面

支持选择多个目标节点，选择执行模式（检查模式/执行模式），选择任务类别（全部/密码策略/系统安全/SSH加固/文件权限）。执行过程中实时显示进度和结果。

3. 报告查看界面

展示历史执行记录列表，支持查看详细报告，包含统计图表（饼图）和每台服务器每个任务的执行结果。

4. 回滚功能

在执行结果页面，每个成功执行的任务显示回滚按钮，点击后可回滚到原始配置状态。

API接口说明

接口路径 方法 说明
/api/nodes GET 获取节点列表
/api/nodes POST 添加新节点
/api/nodes/{host} PUT 更新节点信息
/api/nodes/{host} DELETE 删除节点
/api/nodes/{host}/test POST 测试节点连接
/api/tasks GET 获取任务列表
/api/execute POST 执行任务
/api/tasks/{task_id} GET 获取执行状态
/api/reports GET 获取历史报告
/api/reports/{id} GET 获取报告详情
/api/rollback POST 回滚任务

注意事项

  1. 建议在生产环境执行前，先在测试环境验证
  2. 部分配置修改后需要重启相应服务（如SSH）
  3. 执行前请确保有系统备份
  4. 建议在业务低峰期执行
  5. 部分命令需要root权限
